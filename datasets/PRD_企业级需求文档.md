# AI Listing Generate PRD（企业级）

## 1. 文档信息
- 项目：AI Listing Generate
- 版本：v1.0（基于当前仓库实现反推）
- 日期：2026-04-09

## 2. 产品背景
跨境电商上架存在高频重复工作：类目匹配、标题描述生成、翻译、OCR。当前项目目标是提供统一 AI 服务接口，降低人工成本并提升上架一致性。

## 3. 产品目标
1. 提供统一 API：商品生成、翻译、OCR。
2. 商品生成支持异步任务、轮询、回调通知。
3. 模型与提示词配置可数据化管理（DB 配置）。
4. 构建企业可扩展基础（日志、状态流转、压测基线）。

## 4. 用户与角色
1. 商家运营：提交商品信息并获取可发布文案。
2. 对接系统（ERP/刊登系统）：通过 API 批量调用。
3. 配置运营：维护提示词、模型路由、鉴权配置。
4. 运维/SRE：监控稳定性、容量与告警。

## 5. 功能范围
### 5.1 In Scope
1. 文本翻译：`/api/r1/c/translate`、`/api/r1/c/batchtranslate`
2. 图片 OCR：`/api/r1/c/ocr`
3. 商品异步生成：`/api/r1/shop/ailist`、`/api/r1/shop/ailist/task/{task_id}`
4. 鉴权：`accesskey` + `accesssecret`
5. 场景化：`x-ai-scene`（默认 `default`）

### 5.2 Out of Scope（当前版本）
1. 配置管理后台 UI
2. 分布式任务队列与多 worker 治理
3. 多租户权限中心与审计平台

## 6. 商品生成业务流程（7阶段）
1. 请求解析与校验
2. 源数据获取或预处理（`product_url` 抓取/手工入参）
3. 类目识别（LLM + embedding/hybrid）
4. 生成标题、描述、属性
5. 结果持久化（源表、任务表、目标表）
6. 任务状态更新（`02 -> 03 -> 00/01`）
7. 可选回调通知（失败重试）

## 7. 功能需求（FR）
1. FR-Auth：所有接口必须校验 API Key/Secret。
2. FR-Translate：支持单条和批量翻译，返回 usage。
3. FR-OCR：支持多图 OCR，返回 usage。
4. FR-Shop-Create：创建任务返回 `task_id`、`polling_url`。
5. FR-Shop-Poll：轮询返回状态、模型、生成结果。
6. FR-Callback：支持 `notice_url`，记录通知状态与重试次数。
7. FR-Scene：场景化提示词和模型路由选择。

## 8. 非功能需求（NFR）
1. 可用性：翻译/OCR 可用性 >= 99%（目标）。
2. 性能：并发 20 为现阶段稳定基线；50+ 需架构改造后承诺。
3. 可观测性：日志必须覆盖请求、响应、状态与耗时。
4. 安全：密钥不能硬编码，生产环境禁用测试密钥。

## 9. 验收标准（UAT）
1. 商品任务可创建、可轮询、可回调。
2. 成功任务可返回标题、描述、属性、类目、模型名、usage。
3. 失败任务能给出明确失败状态并可追踪。
4. 提示词/模型配置变更无需改代码即可生效。

## 10. 当前风险与改进项
1. `db_ai_model_route` 在代码和初始化数据中使用，但建表 SQL 未显式创建。
2. `db_category` 在 `create_tables.sql` 末尾存在注释与 DDL拼接风险，需复核初始化可执行性。
3. `db_product_task_detail.usage` 仅 `VARCHAR(128)`，不适合完整 usage JSON。
4. 并发 50/100 压测出现提交与轮询超时，当前 `BackgroundTasks` 有瓶颈。
5. 部分字段类型在 schema 与表结构存在不一致风险（如 `category_id`）。

## 11. 里程碑建议
1. M1：补齐缺失 DDL、字段类型统一、完善错误码。
2. M2：引入队列化执行（Celery/RQ）与回调签名。
3. M3：建设配置后台、任务中心、监控告警与灰度发布。
